## Slack 앱봇알바생 워크스페이스 설정 — 미완료 인수인계

### 상황
- 앱봇알바생 워크스페이스: 생성됨 (team_id: T0ANU3Y83D5? 또는 T0BHLLRA6QH)
- 기존 워크스페이스(kiexpert@naver.com): message_limit_exceeded (무료 플랜 한도)
- WKAppBot 앱: 앱봇알바생에 1개 만들어졌다가 삭제됨 → 다시 만들어야 함

### 필요 작업
1. api.slack.com에서 앱봇알바생 워크스페이스 로그인
   - "Sign in to another workspace" → 앱봇알바생 선택
   - api.slack.com과 app.slack.com 인증이 별도임!
2. Create New App → From an app manifest → 워크스페이스: 앱봇알바생
3. Manifest (YAML):
   display_information:
     name: WKAppBot
     description: Windows App Automation Bot
   features:
     bot_user:
       display_name: WKAppBot
       always_online: true
   oauth_config:
     scopes:
       bot:
         - chat:write
         - chat:write.customize
         - channels:read
         - channels:history
         - app_mentions:read
         - files:write
         - incoming-webhook
   settings:
     event_subscriptions:
       bot_events:
         - app_mention
         - message.channels
     socket_mode_enabled: true
4. Install to Workspace → OAuth 허용
5. Bot Token (xoxb-...) 복사
6. Basic Information → App-Level Tokens → Generate (scope: connections:write) → xapp-... 복사
7. 채널명: #앱봇
8. webhook.json 업데이트:
   W:\SDK\bin\wkappbot.hq\profiles\slack_exp\webhook.json
   → bot_token, app_token, channel, team_id, app_id 교체

### 웹봇 문제 (서제스트 등록됨)
- Chrome --remote-allow-origins=* 미적용 → CDP WebSocket 403
- web read가 navigate해서 모달 날려먹음
- UIA scope 매칭: 다른 Chrome 윈도우 잡음 (VS Code, Slack desktop 등)
- Core cold-start 30초 (FlaUI/UIA)
- 크롬 자동 최소화 (CdpClient.Actions.cs SW_MINIMIZE)
