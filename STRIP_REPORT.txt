CONTEXT & NOTES
===============

The Vision and Android projects implement:
  - VisionCache: Experience database cache for OCR results
  - VisionAnalyzer: Claude Vision API integration for screenshot analysis
  - SimpleOcrAnalyzer: Windows.Media.Ocr wrapper
  - OcrSegment: Model for detected UI elements
  - AdbClient: Android device automation (adb shell integration)

WKAppBot.Core's ActionExecutor uses a 5-tier fallback chain:
  1. UIA (Accessibility) - always available
  2. Vision Cache (optional, now removed)
  3. Simple OCR (optional, now removed)
  4. Vision API (optional, now removed)
  5. Coordinate (always available)

The vision tiers are OPTIONAL and behind runtime flags (_ctx.VisionEnabled), 
so disabling them should be clean if the abstraction is properly refactored.
