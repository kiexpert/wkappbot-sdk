> **Question**: [G:1774875052.115559] We have a Windows automation tool that controls background windows focuslessly (no focus steal). We need to show a REAL-TIME magnified view of the target window, even when it's completely occluded by other windows in front of it.

Current approach: PrintWindow() snapshot → PNG encode → WPF Image decode → display. Problem: it's a one-shot snapshot, not live.

Three candidates for real-time live view of occluded windows:

1. Windows Magnification API (magnification.dll): MagSetWindowSource + MagSetWindowTransform. GPU-accelerated. BUT: MagSetWindowSource uses screen coordinates - if target window is behind another window, it captures the FRONT window's pixels, not the target. Does NOT solve occlusion.

2. DWM Thumbnail API (DwmRegisterThumbnail): Registers a live GPU thumbnail of any HWND, rendered directly by DWM compositor. Works even when window is fully occluded or minimized. No intermediate bitmap. True real-time. But: thumbnail is READ-ONLY display, and requires a destination HWND to render into (needs HwndHost in WPF).

3. Periodic PrintWindow polling: Keep the current approach but call PrintWindow every 100-200ms in a background thread → update WPF Image repeatedly. Works perfectly for occluded windows (PrintWindow ignores occlusion). Simpler, but CPU cost of repeated PNG encode/decode.

Question: Which approach best achieves real-time live view of occluded windows? Is DWM Thumbnail actually the right API here? Are there other approaches (DwmGetDxSharedSurface, SetWindowCompositionAttribute, etc.)? What are the actual tradeoffs?
